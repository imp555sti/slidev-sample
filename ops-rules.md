# Slidev Sample 運用ルール

## 1. VS Code拡張（最小セット）

1. Markdown編集支援拡張
2. ESLint
3. Prettier
4. GitHub Copilot

目的は「個々人の好み」ではなく、「全員が同じ書式・同じ見え方で編集できる状態」を作ることです。

## 2. Copilot利用ルール

Copilotは次の用途に限定します。

1. 章立ての初稿作成
2. 箇条書きの要約
3. 説明文の言い換え

Copilotを次の用途には使いません。

1. フォント/レイアウト方針の最終判断
2. 提出前品質の最終判定
3. 版番号や承認履歴の確定

## 3. レビュー時の確認項目

1. 1ページ1メッセージになっているか
2. 表と本文の情報密度が過剰でないか
3. 日本語フォントの崩れがないか（100%/125%）
4. Copilot生成文を人間が確認済みか

## 4. 完了条件

1. `build:02` と `export:02` が通る
2. 変更差分を説明できる
3. PDF確認観点のチェック結果を記録できる
